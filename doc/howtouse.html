How to use

ebiflyは独自のサーバアプリケーションを起動し、それにクライアントとコンソールが接続する形で動作します。
軽いスクリプトなので、専用のサーバなどを用意する必要はなく、実際に開発を行っているPCで実行するのがおすすめです。
あまり想定はしていませんが、デフォルトでは8080番ポートを使用しますので、外部からアクセスする場合には解放が必要です。

またebiflyはNode.jsとsocket.ioを利用しているので、Node.jsとsocket.ioが必要です。
動作させる環境にNode.js、またはsocket.ioがインストールされていない場合は、 下の方に書いてある手順を参考にインストールを行ってください。

サーバの起動

サーバアプリケーションは /server/ebiserver.js です。
これをNode.jsのアプリケーションとして起動して、サーバの起動は終了です 

cd server
node ebiserver.js

クライアントの接続

クライアントで、デバッグ対象となるページで以下の様に必要なスクリプトを読み込みます。 

<script src="./socket.io.js"></script>
<script src="./ebifly.js"></script>

socket.io.jsは /socket.io 以下に含まれているファイルを使用するか、 socket.ioインストール時に追加されたファイルを使用してください。
準備が整ったら、次のスクリプトのlocalhostをサーバアプリケーションが起動しているPCのIPアドレスやホスト名に置き換えて、 サーバに接続します。 ポート番号を変更をした場合は、ポート番号の書き換えも行ってください。 

ebifly.connect( { host : "localhost", port : 8080});

コンソールへのアクセス

サーバとコンソールを操作するPCが同じ場合、以下にアクセスします。
サーバを別の場所で起動している場合は、localhostをサーバのIPアドレスやホスト名に置き換えてください

http://localhost:8080/console
